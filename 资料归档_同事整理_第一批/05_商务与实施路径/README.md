## 放在这里的原因

这份资料虽然技术含量很高，但它的管理价值更强：

- 定义算法公司的职责边界
- 说明哪些工作不应默认由算法公司负责
- 明确试点交付物、验收口径和不建议接受的交付方式

## 当前可提炼的结论

- 算法公司负责端侧识别、样本闭环、评估和迭代能力
- 算法公司不应默认负责 SAP、主数据治理、完整后台开发和整体实施
- 第一阶段验收不能只看总体准确率，要看离线能力、需复核策略、误判分析和版本管理
