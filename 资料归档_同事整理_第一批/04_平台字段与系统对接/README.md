## 放在这里的原因

这两份资料都在回答“系统怎么承载”和“能力应该放在哪里”：

- 二维码应该承载批次和索引信息，而不是承载图片本体
- AI 复核、样本库、模型发布不应塞进 SAP
- 更合适的结构是 PAD 离线识别 + 独立业务后台 / AI 中台 + SAP 接最终结果

## 当前可提炼的结论

- 二维码适合承载批次 ID、图片 ID、地块编号、时间、端侧结果、置信度、哈希值等索引字段
- 图片本体必须单独存储
- SAP 只接最终确认后的业务结果
- 第一阶段不建议为了 AI 复核专门上重型本地主机
