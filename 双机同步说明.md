# 双机一键同步说明

以后你在 `Mac` 和 `Windows 台式机` 上，都尽量用仓库根目录里的同步脚本来切换工作。

## 脚本位置

- Mac：`一键同步_mac.command`
- Windows：`一键同步_windows.bat`

## 最简单用法

### 在任意一台电脑开始工作前

先点一次本机的一键同步脚本。

它会自动做这几件事：

1. 检查当前项目是不是 Git 仓库
2. 如果本机有改动，先帮你保存成一次提交
3. 拉取 GitHub 上别人或另一台电脑最新的内容
4. 再把本机内容推送回 GitHub

### 在这台电脑做完一轮工作后

再点一次本机的一键同步脚本。

这样另一台电脑下次开始前，只要也点一下它自己的同步脚本，就能拿到最新内容。

## 你的日常操作可以简化成

- 开始前点一次
- 结束后点一次

## 脚本会问你的唯一一件事

如果它发现你本机有改动，会让你输入一次“提交说明”。

这一句不用写得很复杂，比如：

- 更新待办清单
- 调整平台方案
- 补充采样计划
- 完善商务推进内容

如果你直接回车，它也会用默认说明继续同步。

## 什么时候不要继续点另一台电脑

如果脚本提示下面这类情况，就先停下来找我：

- 拉取时出现冲突或异常
- 推送失败
- 当前不在正常分支上
- 这里不是 Git 仓库

## 一个重要提醒

脚本不是“实时自动同步”，而是“你点一下就自动完成一次同步流程”。

也就是说：

- 不用每次都来跟我对话
- 但两台电脑切换前后，最好都点一下脚本
