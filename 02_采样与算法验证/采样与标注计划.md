# 棕榈果一期 AI 分级计数采样与算法验证计划

## 1. 适用范围

本文件用于印尼聚龙集团棕榈果一期 AI 分级计数项目的评估与试点阶段，重点覆盖：

- 园区现场样本采集
- 一类、二类果实分级口径确认
- 图片与视频标注规则
- 模型训练、测试与验收口径
- 手持终端现场识别效果验证

本文件不展开平台功能、运输调度、商务报价等内容。涉及上传、缓存、补传的要求，仅作为样本闭环和算法验证所需的现场前提。

## 2. 当前已确认前提

- 园区网络覆盖率约为 60%，算法验证不能依赖全程在线。
- 一期先按一类、二类果实跑通业务闭环。
- 项目将与园区协同开展现场样本采集。
- 项目需要同步验证手持终端的拍照、离线、本地推理和续航表现。
- 原始图片必须保留，用于追溯、复核、训练迭代和争议处理。
- 识别准确率短期目标参考为不低于 95%，长期目标可向 99% 逼近，但必须以样本积累和持续训练为前提。

## 3. 阶段目标

### 3.1 评估阶段目标

1. 明确一类、二类果实的业务分级标准。
2. 建立第一版棕榈果图像样本库。
3. 完成第一版标注规范和质检规则。
4. 完成训练集、验证集、测试集拆分。
5. 在真实园区环境中验证手持终端拍照与本地识别效果。
6. 给出第一版算法测试报告和是否进入试点的判断。

### 3.2 一期试点目标

1. 支持一类、二类果实目标检测、计数和分类。
2. 支持堆放果串场景的现场拍照识别。
3. 支持本地推理、无网保存测试样本、有网后补传样本与结果。
4. 对晴天、阴天、雨后、强光、弱光、遮挡、堆叠等主要场景形成测试结论。
5. 建立可持续补样、复核、重训、回归测试机制。

## 4. 分级口径确认

一类、二类果实的定义必须由园区业务人员、工厂验收人员和算法团队共同确认。未经确认前，不应把模型输出作为最终验收标准。

### 4.1 分级口径确认项

| 项目 | 需确认内容 | 建议确认人 |
| --- | --- | --- |
| 一类果实定义 | 外观特征、成熟度、颜色、完整度、是否含轻微损伤 | 园区主管、工厂验收人员 |
| 二类果实定义 | 与一类的差异边界、缺陷类型、成熟度下限或上限 | 园区主管、工厂验收人员 |
| 不计入对象 | 空果串、严重遮挡果串、非棕榈果物体、无法判断目标 | 园区主管、算法负责人 |
| 边界样本处理 | 一类/二类难以区分时的归类规则 | 园区主管、工厂验收人员、算法负责人 |
| 争议复核流程 | 标注员无法判断或专家意见不一致时如何处理 | 项目负责人 |

### 4.2 建议输出

- `一类二类果实分级确认单`
- `典型样例图册`
- `边界样本图册`
- `不参与训练样本示例`

典型样例图册至少包含每类 30 张以上高清示例图，并附业务人员确认意见。边界样本图册不建议直接作为第一轮训练主样本，应先用于统一标注口径。

## 5. 样本采集计划

### 5.1 采样原则

- 先保证真实现场样本，再使用数据增强。
- 先覆盖一类、二类核心差异，再扩展复杂场景。
- 先建立清晰样本，再逐步加入遮挡、堆叠、弱光、雨后等困难样本。
- 每张图片必须绑定采集信息，不能只保存裸图。
- 训练集、验证集、测试集必须按园区、时间、场景拆分，避免同一批高度相似图片同时进入训练和测试。

### 5.2 首轮样本量建议

以下为评估阶段建议值，用于启动第一轮模型训练与验证，不作为最终训练规模承诺。

| 样本类型 | 建议数量 | 用途 |
| --- | ---: | --- |
| 一类果实清晰图片 | 1500-2500 张 | 建立一类基础识别能力 |
| 二类果实清晰图片 | 1500-2500 张 | 建立二类基础识别能力 |
| 一类/二类混合堆放图片 | 1000-2000 张 | 验证真实计数和分类 |
| 边界样本图片 | 300-600 张 | 统一口径、后续增强模型 |
| 困难场景图片 | 800-1500 张 | 验证鲁棒性 |
| 视频素材 | 每类/场景若干段，每段 30-120 秒 | 抽帧补样、分析拍摄稳定性 |

首轮建议目标为 5000-8000 张可用图片。若现场差异较大，应优先增加园区、地块、天气、时间段覆盖，而不是只在单一场地重复拍摄。

### 5.3 采样覆盖维度

| 维度 | 必采内容 |
| --- | --- |
| 园区与地块 | 至少覆盖多个地块，避免只采单一来源果串 |
| 时间 | 上午、中午、下午、傍晚 |
| 天气 | 晴天、阴天、雨后，高湿环境下的果串状态 |
| 光照 | 顺光、侧光、逆光、树荫、强反光 |
| 摆放 | 单串、少量堆放、大量堆放、车斗边缘、地面堆放 |
| 距离 | 近景、中景、完整堆放视角 |
| 角度 | 正面、侧面、俯拍、斜拍 |
| 背景 | 泥地、草地、车斗、篷布、工厂地面 |
| 干扰 | 工具、人员手脚、叶片、泥土、其他杂物 |
| 质量 | 清晰、轻微模糊、局部遮挡、雨水附着 |

### 5.4 拍摄要求

- 图片应尽量使用拟投入测试的手持终端拍摄。
- 每次采集应记录终端编号、采集人、时间、园区、地块、天气、网络状态。
- 同一堆果串建议拍摄 3-5 张不同角度图片。
- 图片中果串应尽量完整入画，避免大面积裁切。
- 困难场景可以采集，但必须在样本台账中标记，不能与标准清晰样本混淆。
- 允许采集视频，但训练样本抽帧时应去除高度重复帧。
- 水印不应遮挡果串主体；如加水印，应固定在边角区域。

### 5.5 不合格样本

以下样本不建议进入第一轮训练集，可单独存档用于问题分析：

- 严重失焦或运动模糊，人工也无法判断。
- 果串主体被遮挡超过 50%。
- 图片过暗、过曝，无法区分一类/二类特征。
- 只拍到局部纹理，无法确认是否为完整果串。
- 园区、地块、时间等关键采集信息缺失。
- 标注人员与复核人员无法达成一致的争议样本。

## 7. 标注规范

### 7.1 标注对象

一期只标注棕榈果/果串的一类、二类目标。非棕榈果物体不标注，除非后续引入干扰物类别。

建议类别：

- `class_1_fruit`
- `class_2_fruit`
- `uncertain_fruit`
- `ignore_region`

`uncertain_fruit` 仅用于口径不明或边界样本管理，不建议直接计入第一轮准确率验收。`ignore_region` 用于严重遮挡、无法判断、明显不参与计数的区域。

### 7.2 标注方式

首轮建议采用目标检测框标注。若堆叠严重导致检测框难以稳定表达，应抽取部分样本补充分割标注，用于评估后续是否需要实例分割路线。

标注要求：

- 每个可识别果串单独标框。
- 标框应贴合果串外轮廓，不应包含过多背景。
- 堆叠情况下，只标注可见且可判断类别的果串。
- 同一果串只标一次，不因局部可见区域重复计数。
- 被遮挡但仍可判断为完整目标的，可以标注可见区域外接框，并在备注中标记遮挡。
- 无法判断类别但确认是果串的，标为 `uncertain_fruit`。
- 无法判断是否为果串的，不标注或放入 `ignore_region`。

### 7.3 分类规则

| 情况 | 标注处理 |
| --- | --- |
| 明确符合一类标准 | 标为 `class_1_fruit` |
| 明确符合二类标准 | 标为 `class_2_fruit` |
| 一类/二类边界不清 | 标为 `uncertain_fruit`，进入专家复核 |
| 遮挡严重但类别可判断 | 标对应类别，并标记困难样本 |
| 遮挡严重且类别不可判断 | 标为 `ignore_region` 或不参与训练 |
| 空果串或非目标物 | 不标注，必要时记录为干扰物 |

### 7.4 标注质检

- 每批样本至少 20% 进入复核。
- 边界样本、困难样本、测试集样本必须 100% 复核。
- 标注员与复核员不一致时，由业务专家最终判定。
- 每批输出类别分布、漏标率、错标率和退回率。
- 测试集冻结后不得随意修改；如发现错误，应记录修订版本。

## 8. 数据集拆分

建议拆分比例：

- 训练集：70%
- 验证集：15%
- 测试集：15%
- 现场留出集：按批次单独保留，用于模拟新地块、新天气、新设备效果

拆分规则：

- 同一堆果串的多角度图片不得同时进入训练集和测试集。
- 同一视频抽帧样本不得跨训练集和测试集混用。
- 测试集必须覆盖不同园区、地块、天气、光照和摆放方式。
- 困难样本应在训练集和测试集中都有保留，但比例需单独记录。
- 现场留出集不参与调参，只用于阶段性验收。

## 9. 算法验证指标

### 9.1 检测指标

| 指标 | 说明 |
| --- | --- |
| Precision | 识别出来的果串中，有多少是正确目标 |
| Recall | 实际存在的果串中，有多少被识别出来 |
| mAP | 目标检测综合指标，按一类、二类分别统计 |
| False Positive | 把非目标或错误类别识别成果串 |
| False Negative | 漏识别果串 |

### 9.2 分类指标

| 指标 | 说明 |
| --- | --- |
| 一类准确率 | 人工标为一类的果串中，模型识别正确比例 |
| 二类准确率 | 人工标为二类的果串中，模型识别正确比例 |
| 混淆矩阵 | 统计一类误判为二类、二类误判为一类的情况 |
| 边界样本通过率 | 对业务边界样本的识别一致性，单独统计，不作为首轮硬性验收 |

### 9.3 计数指标

| 指标 | 说明 |
| --- | --- |
| 总数误差率 | `abs(模型总数 - 人工总数) / 人工总数` |
| 一类数量误差率 | `abs(模型一类数 - 人工一类数) / 人工一类数` |
| 二类数量误差率 | `abs(模型二类数 - 人工二类数) / 人工二类数` |
| 图片级准确率 | 单张图片总数和分类数量均在允许误差内的比例 |
| 批次级准确率 | 同一采集批次汇总数量在允许误差内的比例 |

### 9.4 现场终端指标

| 指标 | 建议口径 |
| --- | --- |
| 单张本地推理耗时 | 从拍照完成到显示结果的时间 |
| 连续识别稳定性 | 连续拍摄 100 张是否崩溃、卡顿或丢图 |
| 离线识别成功率 | 无网状态下能否完成拍照、识别、保存 |
| 样本保存完整率 | 图片、识别结果、时间、地点、设备编号是否完整 |
| 补传一致性 | 离线样本恢复网络后是否重复、漏传或错传 |
| 电池表现 | 半天或一天现场采样后的剩余电量 |
| 成像可用率 | 现场拍摄图片中，可进入标注或测试的比例 |

## 10. 准确率验收思路

一期不建议只用单一“准确率”作为验收口径，应按检测、分类、计数和现场可用性分别判断。

### 10.1 建议验收口径

| 验收项 | 建议目标 | 说明 |
| --- | ---: | --- |
| 一类/二类综合识别准确率 | 不低于 95% 参考目标 | 需基于冻结测试集统计 |
| 图片级总数误差 | 建议控制在可业务接受范围内 | 具体阈值需与园区确认 |
| 批次级总数误差 | 应优于单图误差 | 用于模拟实际收果汇总 |
| 一类/二类混淆率 | 单独统计 | 判断分级口径是否稳定 |
| 困难场景准确率 | 单独列示 | 不应掩盖在总体准确率中 |
| 本地推理耗时 | 满足现场连续拍照使用 | 以真实手持终端测试为准 |
| 离线样本完整率 | 接近 100% | 弱网前提下的关键指标 |

### 10.2 测试集要求

用于验收的测试集必须冻结，并满足：

- 不少于 1000 张有效图片，或由双方确认更适合的试点规模。
- 一类、二类数量相对均衡；如业务天然不均衡，应分别报告每类指标。
- 覆盖至少 3 类光照条件、3 类摆放方式和多个地块来源。
- 包含一定比例困难样本，并单独统计结果。
- 所有测试图片必须经过人工复核。

### 10.3 报告方式

算法测试报告至少包含：

- 测试数据来源和采集时间
- 测试集规模、类别分布、场景分布
- 检测、分类、计数指标
- 一类/二类混淆矩阵
- 典型正确样例
- 典型错误样例
- 困难场景单独分析
- 手持终端现场表现
- 问题原因判断
- 下一轮补样和优化建议

## 11. 现场测试流程

建议按以下流程组织首次现场测试：

1. 园区确认一类、二类分级标准和样例。
2. 选择 2-3 个地块作为首轮采样点。
3. 使用拟测试手持终端进行现场拍照与视频采集。
4. 每个采样点记录天气、光照、网络状态和操作人员。
5. 现场同步人工记录一类、二类数量，作为抽样对照。
6. 无网或弱网场景下验证本地识别和样本保存。
7. 回到有网环境后检查补传数据和原始图片完整性。
8. 标注团队按规范完成标注和复核。
9. 算法团队完成训练、验证和测试。
10. 输出测试报告，并形成下一轮补样清单。

## 12. 主要风险与应对

| 风险 | 表现 | 应对动作 |
| --- | --- | --- |
| 分级标准不统一 | 同一果串业务人员判断不一致 | 先做分级确认单和样例图册 |
| 样本来源单一 | 测试时换地块效果下降 | 首轮即覆盖多个地块和多种场景 |
| 困难样本不足 | 雨后、弱光、遮挡场景误差高 | 单独设困难样本采集任务 |
| 标注质量不稳 | 漏标、框偏、类别错 | 建立复核、退回和抽检机制 |
| 测试集污染 | 测试结果虚高 | 按批次和场景拆分，冻结测试集 |
| 手持成像不稳定 | 模糊、过曝、夜间不可用 | 同步记录设备型号和成像可用率 |
| 弱网影响闭环 | 样本丢失或重复 | 离线保存完整率和补传一致性必须测试 |
| 95%口径不清 | 各方对准确率理解不同 | 拆分检测、分类、计数、现场指标 |

## 14. 待确认问题

1. 一类、二类果实的正式业务定义是什么，是否与工厂验收标准完全一致。
2. 一期识别对象是单个果串、散果，还是二者都包含。
