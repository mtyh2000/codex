# 对话板块同步库

这个仓库用于把不同对话线程里的长期有效内容同步到 GitHub，形成一个可持续维护的工作知识库。

## 当前板块

- `00_工作底库`：长期工作背景、客户线、表达偏好、协作方式
- `01_项目专区`：所有项目工作的总入口和项目管理规则
- `02_PPT专区`：PPT 汇报逻辑、审美、提示词和页面优化标准
- `03_Word-Excel-PDF专区`：文档、表格、PDF 等办公文件处理规则

## 使用原则

- 对话里出现长期有效的信息，要及时沉淀到对应板块。
- 不同客户、项目、材料类型要分区保存，避免混在一起。
- 面向客户的内容重视价值、场景、路径和可信度。
- 面向内部的内容重视任务、字段、流程、责任人、时间节点和风险。
- PPT 内容默认追求清楚、简洁、科技感、实例化和高级感。

## 后续维护方式

每次对话里形成新的长期规则、客户资料、项目结论、PPT审美方法或常用提示词，都可以更新到这个仓库，再同步到 GitHub。
