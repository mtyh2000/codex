# 对话归档说明

## 这是什么

这个目录用于统一存放“零散对话里真正有用的内容”。

这里不追求保存每一句聊天原文，而是把值得保留的信息沉淀成文档，方便：

- 两台电脑同步
- 后续检索
- 避免同样的问题反复聊
- 把碎片对话变成可积累的知识

## 重要说明

### 可以同步的

- 你手动整理出来的 Markdown / TXT / 图片
- 我帮你整理出来的对话摘要、结论、清单、模板
- 放进这个仓库里的文件

### 不适合直接同步成文件夹的

- App 里的原始聊天窗口本身
- 没有整理过的长篇来回对话

更稳的做法是：

把每次对话中“真正值得保留的内容”整理成一页文档，再放进这里。

## 目录建议

- [01_待整理](01_待整理)
  先把零散内容丢进来，后面再整理。

- [02_已整理_项目相关](02_已整理_项目相关)
  虽然不是主项目文档，但和项目推进有关系的聊天结论。

- [03_已整理_通用咨询](03_已整理_通用咨询)
  偏通用的问题，比如同步、工具使用、工作流建议等。

- [99_模板](99_模板)
  存放归档模板，方便后续快速记录。

## 建议用法

### 情况 1：一段对话很有价值

做法：

- 新建一份 Markdown
- 写清楚主题、背景、结论、下一步

### 情况 2：只是零碎想法

做法：

- 先放进 `01_待整理`
- 以后再归类

### 情况 3：和当前项目强相关

做法：

- 如果已经形成正式口径，直接写入主文档
- 如果还只是中间讨论，就先放到 `02_已整理_项目相关`

## 推荐命名方式

建议文件名尽量清晰，不要只写“聊天记录1”。

推荐：

- `2026-06-12_双机同步方案.md`
- `2026-06-12_果质分类讨论摘要.md`
- `2026-06-12_PAD离线识别口径.md`

## 维护原则

- 每次只保留有价值内容
- 一条对话尽量整理成一页
- 能写结论就不要只贴原文
- 能归入正式项目文档的，最终还是尽量回收到正式文档中
